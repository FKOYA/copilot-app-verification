# sandbox/ について

ここは Copilot アプリ(エージェント)に実際に触らせる検証用コードを置く場所。Verification-checklist.md のパターン①〜③(ローカル未初期化 / ローカル git init 済み / リモート接続済み)を試すための最小限のダミープロジェクトを想定している。

## 想定する構成例

```
sandbox/
├── README.md                 # このファイル
├── pattern2-local-repo/      # ②: git init 済み・リモート未接続のダミープロジェクト
│   ├── .git/
│   └── (適当な言語のトイプロジェクト。例: 簡単な CLI や API の雛形)
└── pattern3-remote-repo/     # ③: このリポジトリ自身、または別途 GitHub 上に用意した検証用リポジトリへの案内
    └── NOTE.md                # 実体は GitHub 上の別リポジトリの場合、リンクだけ記載
```

- **パターン①(git init なし)** はディレクトリを都度その場で作って消す使い捨てのケースが多いため、常設のサブディレクトリは用意していない
- **パターン②** はエージェントに実際にコードを書かせて壊れても問題ないよう、小さく無関係な内容にする(実プロジェクトのコードは絶対に置かない)
- **パターン③** は GitHub 連携の検証が目的なので、public/private どちらかの実 GitHub リポジトリが必要になる場合が多い。このリポジトリ自体を使う、または検証専用の別リポジトリを用意し、ここには参照だけ残す

## 運用ルール

- ここで作ったブランチ・PR はマージせずクローズして証跡として残す(README.md「6. 運用ルール」と同じ)
- 検証用コードは使い捨て前提。壊れてもよい・消しても惜しくない内容に留める
